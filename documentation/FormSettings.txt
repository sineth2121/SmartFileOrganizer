1. PURPOSE
   - Comprehensive settings management for all application behavior
   - Manages 13 settings stored as key-value pairs in the app_settings table
   - Includes database credential management, file operation defaults,
     exclusion lists, and startup behavior toggles
   - Schema migration: handles upgrading from old single-column schema
   - Connection testing and history cleanup utility buttons

5. KEY METHODS

   private void EnsureSettingsTable() (line 30)
   - Schema migration logic:
     * Checks if old schema exists (column 'destination_folder_path' in
       app_settings table)
     * If old schema:
       - Creates app_settings_new table with key-value schema
       - Migrates destination_folder_path value
       - Renames app_settings → app_settings_old
       - Renames app_settings_new → app_settings
       - Drops app_settings_old
     * If new schema already exists:
       - Creates table if not exists with key-value schema
   - The old schema was a single-row table with columns:
     destination_folder_path, created_at
   - The new schema is: id, setting_key (UNIQUE), setting_value, updated_at

   private string GetSetting(string key, string defaultValue) (line 87)
   - SELECT setting_value FROM app_settings WHERE setting_key = @key
   - Returns defaultValue if no result

   private void SetSetting(string key, string value) (line 102)
   - INSERT INTO app_settings (setting_key, setting_value, updated_at)
   - ON DUPLICATE KEY UPDATE setting_value = @value, updated_at = NOW()

   private void LoadSettings() (line 118)
   - Calls EnsureSettingsTable() first
   - Reads each setting by key and populates the corresponding control
   - Number settings are clamped: nudMaxFileSize (0-99999),
     nudDbPort (1-65535)
   - Checkbox settings: chkConfirmActions, chkVerifyFiles, chkScanDups

9. ARCHITECTURE NOTES
   - Settings are persisted as key-value pairs for flexibility
   - The schema migration ensures backward compatibility
   - DatabaseConfig.cs uses a hardcoded connection string and is NOT
     dynamically updated from these settings — the settings here are only
     used for the Test Connection button
   - The connection testing creates its own MySqlConnection rather than
     using DatabaseConfig.GetConnection()
