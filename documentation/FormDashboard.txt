1. PURPOSE
   - Acts as the main application window / navigation hub
   - Contains a header, a sidebar with navigation buttons, and a panelMain that
     hosts all child forms (embedded as non-top-level docked controls)
   - The first form created by Program.cs: Application.Run(new FormDashboard())

2. FIELDS & PROPERTIES
   - private Form activeForm = null;
     Tracks the currently visible child form in panelMain.
   - private FormFolderSelection folderSelectionForm = null;
   - private FormConfigureRules configureRulesForm = null;
   - private FormPreviewOrganize previewOrganizeForm = null;
   - private FormOperationHistory operationHistoryForm = null;
   - private FormDuplicateFinder duplicateFinderForm = null;
   - private FormSettings settingsForm = null;
   - private FormDashboardAnalytics dashboardAnalyticsForm = null;
     Lazy singleton references to each child form (created once, reused).

3. CONSTRUCTOR
   - public FormDashboard() => calls InitializeComponent()

4. EVENTS / EVENT HANDLERS

   FormDashboard_Load (line 46)
   - Called on startup
   - Invokes ShowDashboard() which displays FormDashboardAnalytics

   btnFolderSelection_Click (line 94)

   - Sets label2.Text = "📥 Import Files"

   btnDuplicateCleaner_Click (line 114)

   - Sets label2.Text = "🔍 Find Duplicates"

   - Sets label2.Text = "⚡ Preview & Organize"

   - Sets label2.Text = "🕒 Operation History"

   - Lazy-creates FormConfigureRules, opens in panelMain

   btnSettings_Click (line 200)

5. KEY METHODS

   private void OpenChildForm(Form childForm) (line 29)
   - Hides the current activeForm
   - Configures childForm: TopLevel=false, FormBorderStyle=None, Dock=Fill
   - Adds childForm to panelMain.Controls and shows it
   - This embeds the form as a non-top-level child inside the dashboard

   private void ShowDashboard() (line 51)
   - Lazy-creates FormDashboardAnalytics and wires up its navigation events:
     * NavigateToFolderSelection → btnFolderSelection_Click
     * NavigateToConfigureRules → btnConfigureRules_Click
     * NavigateToDuplicateFinder → btnDuplicateCleaner_Click
   - Adds to panelMain if not already present
   - Calls dashboardAnalyticsForm.RefreshData()
   - Sets label2.Text to "📊 Dashboard"

6. UI CONTROLS (defined in Designer)
   - panelMain: The panel that hosts child forms
   - panelSidebar: The left sidebar containing navigation buttons
   - panelHeader: The top header bar
   - label2: Title label (updated on each navigation)
   - btnFolderSelection, btnDuplicateCleaner, btnScanPreview, btnOperationHistory,
     btnConfigureRules, btnDashboard, btnSettings: Navigation buttons

7. NAVIGATION FLOW
   User clicks sidebar button -> singleton form is created (if null) ->
   activeForm is hidden -> new form is embedded and shown -> label2 is updated

8. CONNECTIONS TO OTHER FORMS
   - References all 8 other forms by type
   - FormDashboardAnalytics navigation events delegate back to the dashboard's
     button click handlers (circular but clean event-based navigation)
   - No direct database interaction; purely a UI shell

9. ARCHITECTURE NOTES
   - Singleton-like pattern: each child form is created once and reused
   - Forms are hidden/shown rather than created/destroyed, preserving state
   - No Dispose() logic when navigating away; forms persist in memory
   - The child forms are NOT standalone windows — they are embedded as
     TopLevel=false, docked, borderless controls inside panelMain
