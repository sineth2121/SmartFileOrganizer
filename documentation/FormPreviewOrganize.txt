1. PURPOSE
   - The core execution engine of the application
   - Applies organization rules to imported files, shows a preview tree,
     then executes the actual file copy/move operations
   - Two phases: Prepare Preview (read-only) and Execute Organize (writes)

7. EVENTS / EVENT HANDLERS

   FormPreviewOrganize_Load (line 126)
   - Empty handler

   btnPreparePreview_Click (line 131)
   - PHASE 1: Preview generation (read-only)
   - Loads base destination path from app_settings
   - Loads all active rules from organization_rules (ORDER BY id ASC)
   - Clears predicted_destination in imported_files
   - Counts items: WHERE file_type='File' OR (file_type='Folder' AND
     is_excluded=1)
   - For each item, iterates through rules in order; first match wins
   - If no match: predicted_destination = baseDestinationPath\Unorganized_Items
   - Updates imported_files.predicted_destination for each item
   - Calls PopulatePreviewTree() to build TreeView
   - Updates stats labels (total, matched, unmatched)

   btnExecuteOrganize_Click (line 618)
   - PHASE 2: File operation execution (writes to disk)
   - Confirms with user (shows Move vs Copy based on operation_type)
   - Loads all items with predicted_destination (same filter as preview)
   - Generates a unique execution_id via GenerateExecutionId()
   - For each item:
     * Files: creates destination directory, performs File.Copy or File.Move
     * Folders: performs Directory.Move or CopyDirectoryRecursive()
     * Logs to operation_history via INSERT
     * Updates imported_files.destination_path
   - Archives processed items to destination_file_data table via
     INSERT ... SELECT
   - Deletes processed items from imported_files

   tvDestinationPreview_AfterSelect (line 490)

   private bool FileMatchesRule(string fileName, string extension,
       DateTime? modifiedDate, long fileSize, Rule rule) (line 321)
   - Tests a file against a rule's conditions
   - ALL conditions are AND'd together (extMatch && ageMatch && keywordMatch
     && sizeMatch)
   - Extension matching:
     * Parses extensions from parenthesized list via ParseExtensions()
     * E.g., "Documents (.pdf,.docx,.txt)" → ["pdf", "docx", "txt"]
     * Removes leading dot, case-insensitive comparison
   - Age matching:
     * Parses age range strings via ParseAgeDays()
     * Examples: "This Month (< 30 Days)" → (null, 30)
     * "Recent (60–180 Days)" → (60, 180)
     * "Archive (>= 180 Days)" → (180, null)
     * Computes age as (DateTime.Now - modifiedDate).TotalDays
   - Keyword matching:
     * fileName.IndexOf(keyword, OrdinalIgnoreCase) >= 0
   - Size matching:
     * For ranges: fileSize >= min && fileSize < max
     * For min only: fileSize >= min
     * For max only: fileSize < max
   - If a condition field is null in the rule, it's considered a match (true)

   private List<string> ParseExtensions(string extCategory) (line 379)
   - Regex: matches parenthesized content: @"\(([^)]+)\)"
   - Splits by comma, trims whitespace, removes leading dots

   private (int? minDays, int? maxDays) ParseAgeDays(string ageDays) (line 394)
   - Three patterns:
     * Range: @"(\d+)\s*–\s*(\d+)" → min, max
     * Less than: @"<\s*(\d+)" → null, max
     * Greater/equal: ">=\s*(\d+)" → min, null
   - Returns named tuple

12. RULE MATCHING DETAILS
    - Rules are evaluated in database ID order (ASC)
    - FIRST matching rule wins (no fallback to subsequent rules)
    - Each condition (ext, age, keyword, size) is independent and nullable
    - If a condition field is null, it's skipped (treated as match)
    - All conditions must match (AND logic)
    - Unmatched files go to baseDestinationPath\Unorganized_Items
