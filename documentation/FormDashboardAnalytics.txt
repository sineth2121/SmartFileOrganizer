1. PURPOSE
   - The "home page" displayed on application startup
   - Shows summary statistics, file type distribution (pie chart), last 7 days
     operation activity (column chart), destination drive info
   - Runs configurable startup checks (file verification, duplicate scanning)

2. FIELDS & PROPERTIES
   - public event EventHandler NavigateToFolderSelection;
   - public event EventHandler NavigateToConfigureRules;
   - public event EventHandler NavigateToDuplicateFinder;
     These events are raised when the user clicks shortcut buttons on the
     dashboard, allowing FormDashboard to catch them and switch navigation.

   - private bool _startupChecksDone = false;
     Ensures startup checks only run once per application session.

   - private static readonly Color[] CardColors = { ... }
     Color array for summary card backgrounds (Blue, Green, Orange, Red, Purple)

   - private static readonly Dictionary<string, string> ExtensionCategories
     Maps file extensions to category names for pie chart grouping:
     * Documents: pdf,doc,docx,xls,xlsx,ppt,pptx,txt,rtf,odt,ods,csv
     * Images: jpg,jpeg,png,gif,bmp,svg,webp,ico,tiff
     * Video: mp4,avi,mkv,mov,wmv,flv,webm
     * Audio: mp3,wav,flac,aac,ogg,wma,m4a
     * Archives: zip,rar,7z,tar,gz,bz2

   private void RunStartupChecks() (line 135)
   - Only runs once (_startupChecksDone flag)
   - Reads scan_verification_at_startup and scan_duplicates_at_startup settings

   private void VerifyDestinationFiles() (line 180)
   - Iterates all records in destination_file_data WHERE destination_path
     IS NOT NULL
   - For each record, checks if File.Exists(path) on disk
   - Collects IDs of missing files, then deletes them from DB
   - Updates progress bar incrementally

   private void ScanDuplicatesAtStartup() (line 244)
   - Full duplicate scan using MD5 hashing of all destination files
   - Loads all destination_file_data records (skips if File.Exists fails)
   - For each file, uses existing hash or computes new MD5
   - Saves missing hashes back to DB (UPDATE destination_file_data SET file_hash)
   - Groups by hash, counts duplicate sets, computes wasted bytes
   - Calls SaveDuplicateStats() to persist results to app_settings

   private void SaveDuplicateStats(int setCount, int dupFileCount, long wastedBytes) (line 351)
   - Persists three settings: last_dup_sets_count, last_dup_files_count,
     last_dup_wasted_bytes
   - Uses INSERT ... ON DUPLICATE KEY UPDATE pattern

   private static string ComputeMD5(string filePath) (line 385)
   - Computes MD5 hash of file content
   - Returns hex string (lowercase, no dashes)
   - Returns null on any IO/crypto exception

   private void LoadDestinationInfo() (line 412)
   - Reads destination_folder_path from app_settings

   private void LoadPieChart() (line 456)
   - Queries all file extensions from destination_file_data
   - Maps each extension to a category using ExtensionCategories dictionary
   - Unmapped extensions go to "Other"
   - Renders a pie chart on chartFilesByType control
   - Shows percentage labels and a right-docked legend
   - Colors: Documents=Blue, Images=Green, Video=Red, Audio=Purple,
     Archives=Orange, Other=Gray

   private void LoadColumnChart() (line 535)
   - Queries operation_history grouped by DATE(performed_at) for last 7 days
   - Fills in missing days with 0 count so all 7 days are shown
   - Renders a column chart on chartOpsLast7Days
   - X-axis: day abbreviations (Mon, Tue, etc.)
   - Y-axis: file count, auto-scaled with 1.2x padding
   - Blue columns with label values

8. DATABASE TABLES USED
   - app_settings: Reads total_files_imported, destination_folder_path,
     last_dup_sets_count, last_dup_wasted_bytes, scan_verification_at_startup,
     scan_duplicates_at_startup; writes duplicate scan stats
   - destination_file_data: Reads file paths for verification and hashing;
     deletes missing entries
   - operation_history: Reads success count and daily operation counts
   - organization_rules: Reads active rule count

9. LOGIC FLOW SUMMARY
   Load -> RefreshData() -> RunStartupChecks (once) -> LoadSummaryCards ->
   LoadDestinationInfo -> LoadPieChart -> LoadColumnChart
   The user can navigate away via the three quick-link buttons.
